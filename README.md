# 🏋️ Interactive Fitness Command Portal (2026 - 2027)

A fully responsive, free static fitness tracking ecosystem hosted on GitHub Pages. Designed for fast mobile navigation, dark-mode tracking, and localized secure metric logging.

## 📂 Repository File Architecture

Ensure your repository contains exactly these files at the root level:
├── index.html          <- Portal Selector Gateway
├── style.css           <- Unified Styling Tokens & Color Themes
├── vaibhav.html        <- Vaibhav's Lean Mass Dashboard
├── akash.html          <- Akash's Fat Loss Dashboard
├── vaibhav_data.csv    <- Vaibhav's Core Metrics History
└── akash_data.csv      <- Akash's Core Metrics History

---

## 🚀 Instant Deployment Guide (Zero Cost)

1. Create a **Public** GitHub Repository named exactly `fitness-handbooks`.
2. Upload the six files detailed above into the root directory.
3. In your repository page, click the **Settings** tab (gear icon).
4. Select **Pages** from the left-side navigation column.
5. Under **Build and deployment -> Branch**, select `main` (or `master`) and hit **Save**.
6. Wait 60 seconds. Your secure live portal URL will appear at the top of the Pages section:
   `https://<your-github-username>.github.io/fitness-handbooks/`

## 📲 Smart Phone Integration Tip
To get the absolute best out of this setup:
1. Open your custom live link on your smartphone's web browser.
2. Tap the browser options menu and select **"Add to Home Screen"**.
3. This creates a dedicated application icon on your phone for instant, full-screen access to your tracking hub.

## 🔄 The 30-Second Daily Logging Loop

Because static web hosts cannot directly modify files on the server for security reasons, this project utilizes a seamless browser-caching design to capture your input data. Follow this simple loop to back up your progress forever:

1. Tap into your personal profile page from your mobile home screen link.
2. Navigate to **Tab 4: Dynamic Analytics & CSV Engine**.
3. Fill out your metric input fields and tap **Commit Data Entry**. (The updated history will render instantly inside your visualization table).
4. Tap **Download Updated CSV** to generate a clean backup file on your device.
5. Open your GitHub repository in your phone's browser, tap on your personal `.csv` history file, click the **pencil icon** to edit, select all text, paste the contents of your downloaded file, and commit. Your long-term records are now locked in permanently!
